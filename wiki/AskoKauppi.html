I've done C/C++ for a _long_ time (over 30% of my life... gosh!) ;) and sincerely know it's not the best of tools for everything. For me, Lua makes the perfect companion for it, covering areas s.a. string and table management where C would end up being a pointer jungle. I want to marry the two (well, they already are).

My Lua contributions:

  • LuaX (http://luaforge.net/projects/luax/) - High-level C/Lua integration API, covering both static and dynamic linkage.
NOTE: LuaX is considered dead, and unsupported at best. See luaSuper et.al. below.

  • Hamster aka SCons/Lua (http://luaforge.net/projects/hamster/) - Lua front-end to SCons build engine (and others)

  • Lumikki (http://luaforge.net/projects/lumikki/) - HTML generator using Lua macro expansion

  • SDL, Serial and other bindings to Lua, and the GluaX binding interface
NOTE: distribution of these awaiting LuaRocks

  • LNUM patch (http://luaforge.net/projects/lnum/) - allowing best of integers and best of floating point, without affecting Lua external interfaces.

  • luaSuper (http://luaforge.net/projects/luasuper/ - syntax modifiable front end (drop in replacement for 'lua' command)
NOTE: luaSuper is still under work in Nov-07

My Lua opinions / complaints:

  • would like to have LNUM patch included
  • would like to have built-in (Lua/C API level) support for family-aware enumerations/bitfields
  • would like my employers to start using Lua (for their own good and joy)
  • the rest can be achieved via luaSuper :)
