Why Create a New Script Manager?

Before you get the wrong idea, Script Manager was not created to eliminate Steve Donovan's extman. If you've never used SciteExtMan before, it is strongly recommended that you check it out. Without it, this file wouldn't exist. Script Manager was created to expand on extman's ideas by adding different elements and functions. In addition, Script Manager was made to be a very user-friendly and smart script manager.

Script Manager Compatibility

Like extman, Script Manager is a lua script manager for SciTE. It enables multiple scripts to capture standard and custom events without interfering with other scripts. Script Manager was created with SciTE v1.75 on Windows ME and I hate it, especially because I can't update to SciTE v1.76. Before you use Script Manager, you need to know what it is and is not compatible with:
  • Script Manager is available on Windows, but not on GTK (unfortunately, I don't have access to GTK, but I would be happy to make it compatible if I had the proper documentation or help).
  • Script Manager doesn't use SciteOther (I was never able to get it to work...Windows ME).
  • Script Manager is fully compatible with extman (unless the script uses scite_WordAtPos, but that's easy to implement).

Installation

Script Manager is just as easy to install as extman. Just set the ext.lua.startup.script to the manager file (ScriptManager.lua). Like extman, a specific directory can be set, except now the property is ext.lua.script.directory. If you have any questions about Script Manager, just look in the documentation, it has just about everything you need to know about Script Manager.

Example Scripts

The example scripts in the 'Manager Hooks' directory are not explained in the documentation. You will notice that the scripts (minus 1) from extman are not included; because they are compatible, you can just move them over if you need them. Although they are explained within the files themselves, here is a little information about the scripts:
  • AutoComplete+.lua: this script enables users to easily change the size of the autocomplete window, and add pixmaps for autocompletion. The properties can be different in separate lexers.
  • CallTip+.lua: this script allows users to change the colors used in the calltip window, these settings can be different in different lexers too.
  • DisableAltF4.lua: simply, this script disables the Alt+F4 key combination.
  • FastFiles.lua: large files can run slow, the script speeds them up.
  • FoldSymbols+.lua: this script allows users to create and use their own folding symbols.
  • QuickOutput.lua: this script enables the user to quickly close and/or clear the output window.
  • SelectStyle.lua: you should already know by the title, however, this script still makes it possible to select text normally.
  • UpdateStatusBar.lua: again, obvious, but this allows the time and date to be shown correctly without switching files or saving.
