Vis Lua
wiki

This is a proposed design for a cross platform editor and debugger fixme for Lua.

  • Motivation for the project is explained below.
  • VisLuaDesign gives details on features to be included.
  • VisLuaImplementation is some notes on implementation issues.
  • VisLuaGlossary lists all confusing terms used.

If you have any comments or would like to join the development of this project you should join the VisLua-dev mailing list at: http://lists.sourceforge.net/lists/listinfo/visuallua-devs

Motivation

There are a number of Lua editors and debuggers fixme in circulation. Some have fallen by the wayside and some are platform specific. The idea is to produce a simple framework for an open source cross platform develop22ment system which will continue to be maintained by the Lua community. It would also be nice to support different locales.

Existing projects

Lua projects from which ideas could be drawn are :-

  • Ldb - Command line driven debugger. Maintained. [1]
  • Houston - GTK debugger. canceled. [2] (broken link)
  • Titmouse - Win32 API debugger. Maintained. [3]
  • LuaDev - Mac OSX project manager and debugger. No longer exists. [4] (broken link)
  • Lua Studio - MFC IDE. Work cancelled. [5]
  • LuaCmd - TecGraf IUP cross platform GUI library (Currently Win32 & Motif). Supports Lua 3.2. Dormant. [6]
  • Stella - Mac OS. Looks dormant. [7]
  • LuaScript - Win32 COM. Has debugging facilities. Maintained. [8] (broken link)
  • EDELua - Win32 embedded. canceled. [9] (broken link)
  • Lash - Win32 shell. Maintained/In development. [10] (broken link)
  • wxLua - cross-platform GUI library bindings. Active [11]

Other projects which could provide inspiration are listed below. There is not really much point listing every editor there is. Features for the editor can be discussed when it is designed and written.

  • IDLE [12]. A syntax highlighted interactive console written in Python [13] and Tcl/Tk (Tkinter module).
  • SciTE [14] is a simple syntax highlighted editor which supports Lua. The same code that is used here could be used in VisLua.
  • TextPad [15] is a very configurable editor. Such flexibility would be nice.

Platforms

Downloadable binaries of Lua are provided for the following platforms [16]. It would be nice to cover as many platforms as possible with any work done.

  • Unix
  • Windows
  • MacOS & OSX
  • RISC OS
  • EPOC
  • BeOS
  • Palm
