Available Commands

You must be a site administrator to use these links.

  • clean up page history
AdminCommand:maintain
  • display orphaned pages
AdminCommand:orphans
  • display cross reference of page links
AdminCommand:links

There are many variations on this command [1]. Here's one that lists all references to the file area:

AdminCommand:links&inter=2&search=Files

  • rename and delete pages (web interface)
AdminCommand:editlinks
  • reset database lock - Use when "Could not get user-ID lock" error starts happening.
AdminCommand:unlock
  • site lock/unlock
AdminCommand:sitelock&set=1, AdminCommand:sitelock&set=0
  • page lock/unlock - You need to edit the URL, substituting pagename and val as appropriate.
AdminCommand:pagelock&id=pagename&set=val
  • edit banned IP's (web interface)
AdminCommand:editbanned
  • display script version
AdminCommand:version
